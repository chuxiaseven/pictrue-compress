# 图片压缩工具 - 技术方案文档

## 1. 技术栈选型

### 1.1 前端技术栈
- **框架**：Vue 3 (Composition API)
- **构建工具**：Vite 5
- **状态管理**：Pinia
- **路由**：Vue Router 4
- **UI 组件库**：Element Plus / Ant Design Vue
- **样式**：CSS3 + SCSS
- **图片处理**：Browser Image Compression / browser-image-compression
- **图表展示**：ECharts / Chart.js
- **工具库**：Lodash

### 1.2 后端技术栈
- **运行环境**：Node.js 18+
- **框架**：Express.js / Koa2
- **图片处理**：Sharp (高性能图片处理库)
- **文件压缩**：Archiver (ZIP 打包)
- **日志**：Winston
- **配置管理**：Dotenv

### 1.3 开发工具
- **代码编辑器**：VS Code
- **包管理器**：pnpm / npm
- **版本控制**：Git
- **代码规范**：ESLint + Prettier
- **类型检查**：TypeScript

## 4. 核心功能实现方案

### 4.1 图片上传流程
```
1. 前端选择/拖拽文件
   ↓
2. 前端校验（格式、大小）
   ↓
3. FormData 构建
   ↓
4. POST /api/upload
   ↓
5. 后端接收文件
   ↓
6. 后端校验
   ↓
7. 保存临时文件
   ↓
8. 返回文件信息（ID、路径、元数据）
```

### 4.2 图片压缩流程
```
1. 前端发送压缩请求
   {
     fileId: "xxx",
     quality: 75,
     mode: "medium",
     resize: { width: 1920, height: 1080, keepRatio: true },
     format: "jpeg",
     stripMetadata: true
   }
   ↓
2. 后端接收请求
   ↓
3. 读取临时文件
   ↓
4. 使用 Sharp 进行处理
   - 解码图片
   - 尺寸调整（如需要）
   - 格式转换（如需要）
   - 质量压缩
   - 元数据去除（如需要）
   - 编码输出
   ↓
5. 保存压缩后文件
   ↓
6. 返回压缩结果
   {
     originalSize: 1024000,
     compressedSize: 350000,
     compressionRatio: 0.34,
     downloadUrl: "/api/download/xxx"
   }
```

### 4.3 批量压缩流程
```
1. 前端发送批量压缩请求
   {
     fileIds: ["id1", "id2", "id3"],
     config: { quality: 75, ... }
   }
   ↓
2. 后端创建压缩队列
   ↓
3. 顺序处理每个文件
   - 读取文件
   - 压缩处理
   - 保存结果
   - 更新进度
   ↓
4. 所有完成后返回结果
   {
     total: 3,
     success: 2,
     failed: 1,
     results: [...]
   }
```

### 4.4 对比功能实现
```
1. 前端展示原图和压缩图
   ↓
2. 用户选择对比方式
   - 并排：左右两个 <img> 标签
   - 滑块：CSS clip-path 或 Canvas
   - 切换：显示/隐藏切换
   ↓
3. 显示对比数据
   - 文件大小
   - 压缩比例
   - 分辨率
```

### 4.5 下载功能实现
```
单张下载：
1. 前端请求 /api/download/:id
   ↓
2. 后端设置响应头
   - Content-Type: image/jpeg
   - Content-Disposition: attachment; filename="xxx.jpg"
   ↓
3. 读取文件并流式返回

批量下载：
1. 前端请求 /api/download/batch
   ↓
2. 后端使用 Archiver 创建 ZIP
   - 添加多个文件
   - 设置 ZIP 压缩
   ↓
3. 流式返回 ZIP 文件
```

## 6. API 接口设计

### 6.1 上传接口
- `POST /api/upload` - 上传单个文件
- `POST /api/upload/batch` - 批量上传

### 6.2 压缩接口
- `POST /api/compress` - 压缩单个文件
- `POST /api/compress/batch` - 批量压缩
- `GET /api/compress/status/:id` - 查询压缩状态

### 6.3 下载接口
- `GET /api/download/:id` - 下载单个文件
- `GET /api/download/batch` - 批量下载

### 6.4 历史接口
- `GET /api/history` - 获取历史记录
- `DELETE /api/history/:id` - 删除历史记录
- `DELETE /api/history` - 清空历史
- `GET /api/history/export` - 导出历史

### 6.5 设置接口
- `GET /api/settings` - 获取设置
- `POST /api/settings` - 保存设置

## 9. 部署方案

### 9.1 开发环境（桌面应用方式）
- 同时启动 Vite 开发服务器和 Electron
- Vite 开发服务器：提供前端热重载开发环境
- Electron：作为桌面应用容器加载 Vite 开发服务器
- 启动流程：
  1. 启动 Vite 开发服务器（默认 http://localhost:5173）
  2. 启动 Electron 应用
  3. Electron 通过 `BrowserWindow` 加载 Vite 开发服务器地址
  4. 后端服务在 Electron 主进程中运行
- 热重载支持：
  - 前端代码修改：Vite 自动热更新
  - 后端代码修改：Electron 自动重启
  - Electron 配置修改：Electron 自动重启
- 调试方式：
  - 使用 Electron DevTools 进行前端调试
  - 使用 VS Code 调试后端 Node.js 服务
  - 使用 Electron 的远程调试功能
- 启动命令：
  - `pnpm run dev` - 同时启动 Vite 和 Electron

### 9.2 生产环境
- 前端构建：`pnpm run build`
- 后端编译：`tsc`
- 使用 Electron Builder 或 Tauri Builder 打包
- 生成可执行文件（.exe）
- 静态文件内嵌到应用中
- 后端服务作为应用的一部分运行

### 9.3 桌面应用打包
- 使用 Electron Builder 打包
- 使用 Tauri Builder 打包
- 生成可执行文件（.exe）
- 支持自动更新

## 10. 技术风险与应对

### 10.1 大文件处理
- 风险：大文件可能导致内存溢出
- 应对：限制文件大小、使用流式处理、分块处理

### 10.2 格式兼容性
- 风险：某些图片格式处理困难
- 应对：使用成熟的 Sharp 库、提供错误提示、降级处理

### 10.3 浏览器兼容性
- 风险：旧浏览器不支持某些 API
- 应对：提供 Polyfill、降级方案、提示用户升级浏览器
